# TRESC 仓库理解速览（中文）

这个文档用于快速说明：TRESC 在做什么、核心算法链路是什么、与常见量化程序相比有什么“特色”。

## 1. 这是一个什么项目

TRESC（Thomas Relativistic Electronic Structure Calculation）是一个面向**非周期分子体系**的电子结构程序，基于 Born–Oppenheimer 近似，支持 HF/KS-SCF，并重点支持**两分量 DKH2 相对论哈密顿量**。主程序使用 Fortran 2008 编写。

## 2. 核心算法主线

1. 读入输入关键词、几何结构（XYZ）和基组（GBS/MOLDEN）
2. 构建哈密顿量（非相对论 Hartree 或 DKH2）
3. 执行 SCF（支持阻尼、DIIS、正交化策略切换）
4. 输出能量、轨道信息以及自旋相关分析

此外，程序支持通过可视化工具链导出网格数据（实空间/动量空间），用于展示复自旋轨道的相位与 SOC 特征。

## 3. 主要“特色”

- **相对论电子结构为默认重点方向**：不仅有 DKH2 1e/2e 项，还提供可选的 SRTP 相关项（`pppVp`）用于探索性研究。
- **可解释性偏强**：不仅给总能量，还强调轨道成分、`<s^2>`、自旋纯态分解、TRS 破缺度量（κ）。
- **可视化生态较完整**：仓库内置 Vis2C 脚本，支持 cub/mog/pro 三类可视化管线。
- **工程取向清晰**：使用直接构建 Fock（省内存但耗时），结合 Schwarz 筛选和 OpenMP 并行，明显是面向“可算大体系”的折中设计。

## 4. 代码组织

- `tkernel/`：主 Fortran 内核（积分、哈密顿量、SCF、表示与网格）
- `vis2c/`：Python 可视化脚本
- `basis/`：大量相对论相关基组库
- `docs/`：算法与数学说明、图示
